Constants

defaultBucketSize → const int
DefaultBucketSize is the Amino DHT bucket size (k in the Kademlia paper). It represents the maximum number of peers stored in each k-bucket of the routing table.
defaultConcurrency → const int
DefaultConcurrency is the suggested number of concurrent requests (alpha in the Kademlia paper) for a given query path in Amino DHT. It determines how many parallel lookups are performed during network traversal.
defaultLookupCheckConcurrency → const int
DefaultLookupCheckConcurrency is the maximal number of routines that can be used to perform a lookup check operation, before adding a new node to the routing table.
defaultMaxLatency → const Duration
DefaultMaxLatency is the maximum acceptable latency for peers in the routing table. Peers with higher latency will not be added to the routing table.
defaultMaxPeersPerIPGroup → const int
DefaultMaxPeersPerIPGroup is the maximal number of peers with addresses in the same IP group allowed in the routing table. Once this limit is reached, newly discovered peers with addresses in the same IP group will not be added to the routing table.
defaultMaxPeersPerIPGroupPerCpl → const int
DefaultMaxPeersPerIPGroupPerCpl is maximal number of peers with addresses in the same IP group allowed in each routing table bucket, defined by its common prefix length to self peer id. also see: DefaultMaxPeersPerIPGroup.
defaultOptimisticProvideJobsPoolSize → const int
DefaultOptimisticProvideJobsPoolSize is the asynchronicity limit for in-flight ADD_PROVIDER RPCs. It's set to a multiple of OptProvReturnRatio * BucketSize.
defaultProviderAddrTTL → const Duration
DefaultProviderAddrTTL is the TTL to keep the multi addresses of provider peers around. Those addresses are returned alongside provider. After it expires, the returned records will require an extra lookup, to find the multiaddress associated with the returned peer id.
defaultProvideValidity → const Duration
DefaultProvideValidity is the default time that a Provider Record should last on Amino DHT before it needs to be refreshed or removed. This value is also known as Provider Record Expiration Interval.
defaultRecordTTL → const Duration
DefaultRecordTTL is the maximum time that any node will hold onto a record from the time it's received. This is the same as DefaultProvideValidity.
defaultRefreshInterval → const Duration
defaultRefreshQueryTimeout → const Duration
DefaultRefreshQueryTimeout is the timeout for routing table refresh queries.
defaultResiliency → const int
DefaultResiliency is the suggested number of peers closest to a target that must have responded in order for a given query path to complete in Amino DHT. This helps ensure reliable results by requiring multiple confirmations.
defaultUsefulnessGracePeriod → const Duration
DefaultUsefulnessGracePeriod is the period after which a peer is considered not useful if it hasn't been useful during this time.
protocolID → const ProtocolID
ProtocolID is the latest protocol identifier for the Amino DHT.
protocolPrefix → const ProtocolID
ProtocolPrefix is the base prefix for Amino DHT protocols.
serverModeMinPeers → const int
ServerModeMinPeers is the minimum number of peers required in the routing table for the DHT to switch to server mode when in auto mode.
